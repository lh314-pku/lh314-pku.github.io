# Multiview Stereo 多目立体

多目立体是对双目立体视觉的扩展，输入多张同一物体/场景的照片（相机已校准），输出对应的三维形状的表示形式。

这些图片可以来自任意数量的视角（从 **2 张到上千张**），使用不同的相机位置（例如，相机网络、视频拍摄等）。相机的校准可以通过 **运动中的结构（Structure from Motion）** 或一些**特殊设备**完成，比如无人机、阵列相机等。

而常见的 3D 表达有：体素、网格、点云……（参考VCL）

基本想法：图像间的密集对应

## Visual Hull based MVS 基于MVS的视觉包络

### Visual Hull

视觉包络的核心思想是：通过从多视角的图像中提取前景（foreground）部分，生成一个物体的粗略三维形状。主要用于三维重建的初步阶段，生成**外轮廓**模型。

将拍摄的照片与单纯的背景图做减法，得到仅包含前景的图片并提取轮廓（silhouette）。但它无法捕捉到复杂的细节或凹陷部分，因为只能按外轮廓构建形状。

通过 **Plane-Sweep Stereo（平面扫描立体重建）**，假设场景中存在多个深度平面（类似于一层层切片），从参考相机的角度出发，将所有输入图像投影到这些深度平面上。

对每个深度平面，计算投影图像在每个像素位置上的方差（variance）。方差越小，说明这些像素在该深度平面上更一致。对于每个像素，选择方差最小的深度值，作为该像素的最终深度。

### Depth Map based MVS

深度图的多视角重建过程：

- **通过相邻视图计算深度图**；（可以使用双目视觉立体匹配的方法来完成这一步）

- **将多个深度图合并成一个整体的三维体积**；（即将这些深度图合并到一个统一的三维空间中）

- **从三维体积中提取三维表面**

对于每个像素，需要根据匹配评分找到最适合的匹配点。此过程中不需要过多的局部正则化（Local Regularizations），可以直接使用两视图立体匹配方法。

**局部正则化**是指在生成深度图时，对深度图进行独立的平滑处理，以减少噪声或异常值。

**早期局部正则化（Early Local Regularization）** 可以补充某一视角下被遮挡部分的深度图，但是可能导致最后的三维表面缺少细节。如果不进行过早的正则化，可以保留更多的细节信息，最终在三维空间中进行优化时，得到更准确的三维表面。

最好的方法是：**在深度图合并到三维体积之后，再进行三维空间的优化和正则化处理**。这样能够更好地保留细节信息，提高三维重建结果的质量。

## Patch-based MVS

**基于Patch的MVS**是一种方法，主要思想是通过**图像的局部特征块（Patch）** 来进行三维重建。

关键点：密集的三维重建需要图像之间具有**密集的对应关系（Dense Correspondence）**。

什么是 **Patch**？Patch 是一个局部的小区域，用于表示三维对象表面的特定部分。包括：

- Position：(x,y,x)，中心点的三维坐标；

- Normal：(nx,ny,nz)，所在位置表面的方向，即法线的朝向；

- Extent：radius，半径，即Patch的影响范围。

在数学上可以视为曲面局部的足够小的切平面，用来近似表示三维表面的一小部分。

Why Patches？

- 灵活性高：可以用于表示物体和场景；

- 可以直接提取 3D 数据，且携带局部特征，不会因全局正则化（如深度平滑）而丢失细节信息（指指点点 -> 点云和网格）

一个 Patch $p$ 在数据结构上可以用如下方法定义：

- 位置 c(p)；

- 法向量 $n(p)$；

- 可见图 $V(p)$；（用于指示哪些图是可以看见这个 patch 的）

- Extent： 通常会设置一个像素大小，比如这里的Patch被定义为在可见图像中大约覆盖**9x9像素**。

### Photo Consistency（光度一致性）

广度一致性 $N(I,J,p)$ 用于衡量一个 Patch $p$ 在图像 $I$ 和 $J$ 中颜色的一致性。其公式为：

$$
N(I,J,p)=\frac{\sum(I_{xy}-\bar{I_{xy}})\cdot
(J_{xy}-\bar{J_{xy}})}
{\sqrt{\sum(I_{xy}-\bar{I_{xy}})^2\sum(J_{xy}-\bar{J_{xy}})^2}}
$$

> 实质上是相关系数的变体计算公式，描述了两个随机变量 $I_{xy}$ 和 $J_{xy}$ 的相关性。

当两张图像中对应区域的颜色变化一致时，光度一致性值会更加接近 1。

对于 Patch $p$ 来说，可以综合多个视图的光度一致性来评估Patch的质量：

$$
N(p)=\frac{\sum_{i=1}^n\sum_{j=i+1}^nN(I_i,J_j,p)}{(n+1)n/2}
$$

Patch 的优化过程是通过微调位置和法向量来提高重建质量：

$$
\{c(p),n(p)\}=\arg\max_{c(p),n(p)}N(p)
$$

称为：**Patch优化**。

而 $V(p)$ 的更新则更加复杂，目的是通过计算光度一致性，筛选出对重建有益的图像，剔除可能引入误差的图像，从而提高多视图三维重建的质量。

- 假设 Patch $p$ 的可见集 $V(p)=\{Image_1,Image_2,Image_3,Inage_4\}$

- 分别计算两两之间的光度一致性，并计算每张图片的总分数；

- 选取得分最高图像作为基准，并设置阈值（例如 0.7）

- 检查其余图像与基准图像的光度一致性，剔除低于阈值的图像；

- 更新 $V(p)$。

至于一个 Patch 是否正确，我们做如下要求：经过 V(p) 更新后可见图像不少于 3 个。如果数量不足，说明Patch的光度一致性不足，可能是由于误匹配造成的，此时应舍弃该Patch。

至此，我们总结 **Patches-based MVS 的步骤**：

- 特征检测 Feature Detection；

- 特征匹配 Feature Matching；

- Patch 扩展和过滤 Patch Expansion and Filtering。

### Feature detection

从输入图像中提取显著特征点，为后续的Patch生成提供基础。

可以提取 Harris 角点检测器（角点）和高斯差分（斑点）的局部极大值。

### Feature Matching

经过特征提取，我们需要得到有哪些 patch ，并对其初始化：

- 借助对极几何（指极线），我们可以将图像 $I_1$ 中的 Patch 区域 在 $I_2$ 找到对应区域。通过三角测量，我们得到初始化的 $c(p)$，并将 $n(p)$ 初始化为 $I_1$ 的深度方向（即 patch 平面与图像平行），将 $I_1$ 和 $I_2$ 加入 $V(p)$。

- 查询其他图片，计算 Patch 在其他图像（例如 $I_3$）上的投影，并计算与 $I_1$ 的光度一致性，将大于阈值的图像加入 $V(p)$；

- 对这个 Patch 做上述优化过程，并进行验证（即 $|V(p)|\ge3?$）

- 如果这个 Patch 可以保留，就把它在各个图像中的投影区域标记为 **Occupied** 以免重复检测。

### Patch Expansion and Filtering

但是在特征提取与匹配时，不一定能涵盖到图像的所有部分，比如一些缺乏纹理的区域，此时需要进行 expansion 以补充这些部分对应的 patch。

我们从已知的Patch扩展：

- 标记当前 Patch 周围的空白像素（未被占用），如果其附近像素均未被占用，说明其并不完备，周围存在空洞，需要扩展；

- 新建 Patch $q$，其 $n(q)$ 与 $V(q)$ 与 $p$ 相同，$c(p)$ 可以设置为 附近空白像素与相机光心连线 与 $p$ 的切平面的交点。

- 初始化后，还需要进行 Patch优化和验证。

扩展完后，我们还需要过滤掉不可靠的 Patch，只保留那些在多个照片中都看起来合理的 Patch。这可以用 **可见一致性（Visibility Consistency）** 衡量：

Idea：Patch $p_1$ 应该只能被那些没有被遮挡的相机看到。

$$
|V(p_1)|\cdot N(p_1)<\sum_{i=2}^6N(p_i)
$$

（$N(p)$ 可以评价一个Patch的质量/置信度，如果 $p_1$ 的可见性与质量之积小于其他之和，则过滤掉 $p_1$）
